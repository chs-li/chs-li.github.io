# 屏蔽惡意發言系統專案總覽

整合大型語言模型（LLM）與傳統機器學習模型，打造多層語意過濾架構，用於偵測並攔截聊天室中的不當或攻擊性訊息。系統兼顧高召回與高精度，協助減少人工審查成本並提升風險控管效率。

## 主要亮點
- **多路線策略**：LLM、語意模型與傳統 ML 共同投票，提升整體準確度與穩定性。
- **多層過濾**：黑名單、語意相似度、檢索方法與分類模型組合，提高阻擋能力。
- **資料管理**：可維護黑白名單、標註資料與關鍵字庫，以因應場景快速更新。
- **前端友善**：提供 API 與風險等級輸出，方便前端整合並設定處置規則。

## 功能概覽
### 歷史方案摘要
- 不同深度的語意模型與傳統 ML 皆曾應用於惡意語句分類。
- 結合黑名單與語意檢索（如向量相似度、文字檢索）提升召回。
- 傳統 ML（如樹模型、邏輯迴歸）用於整合多項訊號，提高穩定性。

### 資料與標註流程
- 維護黑白名單、標註語料與自動生成的補充資料。
- 透過語意相似度協助人工複核，提高資料一致性。
- 平衡正負樣本後再輸入模型訓練，以提升判斷品質。

### 系統迭代策略
- 依需求切換偏高召回或高精度的模型組合。
- 支援快速更新黑名單或規則，使新需求立即生效。
- 可定期以最新標註資料重新評分或微調模型。

### 處理流程（抽象化）
1. 接收訊息  
2. 基礎規則過濾（如黑名單、近似字串）  
3. 語意檢索比對相似語句  
4. 模型或多路線投票產出風險判定  
5. 輸出風險等級與處置建議（阻擋 / 模糊化 / 人工複核）

### 模型與方法搭配（依場景選用）
- **語意模型路線**：支援使用深度模型進行分類或微調，適合要求高精度的場景。
- **檢索路線**：透過關鍵字、文字檢索或向量檢索做初步篩查。
- **模糊比對**：應對錯字、口語變形，提高召回率。
- **傳統 ML 票選**：整合不同訊號，提高整體穩定度。

### 數據資源與維運
- 包含黑白名單、標註資料、未標註語料與關鍵字等資源。
- 支援批次更新與在線更新，方便快速迭代風險規則。

## 使用與部署建議
- **快速上線**：先用規則與檢索搭建基礎架構，再引入深度模型提升精度。
- **召回／精度取捨**：依風險等級選擇多層檢測與模型組合。
- **資料更新**：定期從實際對話收集樣本並更新名單與關鍵字。
- **前端整合**：對外提供風險等級、操作建議與日誌，並支援可調閾值。
